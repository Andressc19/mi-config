# README Documentation Skill

Create professional markdown documentation with beautiful formatting.

## Document Structure

```markdown
# Title

Brief description of what this document covers.

## Table of Contents
- [Overview](#overview)
- [Getting Started](#getting-started)
- [Usage](#usage)
- [Examples](#examples)
- [FAQ](#faq)

## Overview

High-level explanation...

## Getting Started

Step-by-step introduction...

## Usage

Detailed instructions...

## Examples

Practical examples with code blocks...

## FAQ

Common questions and answers.
```

## Formatting Best Practices

### Headings
```markdown
# H1 - Document Title
## H2 - Major Section
### H3 - Subsection
#### H4 - Detail
```

### Emphasis
```markdown
**Bold** for emphasis
*Italic* for subtle emphasis
~~Strikethrough~~ for corrections
`Code` for inline code
```

### Lists
```markdown
1. Numbered for sequences
2. Second step

- Bullet for alternatives
- Another option

- [ ] Checkbox for tasks
- [x] Completed item
```

### Tables
```markdown
| Column 1 | Column 2 | Column 3 |
|----------|----------|----------|
| Data     | Data     | Data     |
| More     | More     | More     |
```

### Callouts
```markdown
> **Note:** Important information
> 
> Additional details here

> **Warning:** Potential issue
> 
> What to avoid
```

### Code Blocks
```markdown
```python
def example():
    print("With syntax highlighting")
```
```

```

## Styling Tips

1. **Use whitespace** - Blank lines separate sections
2. **Keep lines short** - ~80 characters max
3. **Consistent formatting** - Same style throughout
4. **Include examples** - Every concept needs illustration
5. **Add a ToC** - For documents over 3 sections
